# AWS Compute Optimizer が、6種類の追加リソースタイプについてアイドル推奨をサポートしました

## 概要

AWS Compute Optimizer は、Amazon DynamoDB のプロビジョンドテーブル、Amazon ElastiCache（Redis および Memcached ではなく Valkey という表記は原文の誤字として解釈しますが、原文通り「Valkey」表記を維持します）、Amazon MemoryDB、Amazon DocumentDB（プロビジョンドおよびサーバーレス）、Amazon WorkSpaces、そして Amazon SageMaker エンドポイントのアイドルリソースを特定できるようになりました。これにより、より多くの AWS 環境全体で未使用リソースを検出し、潜在的なコスト削減を識別できます。

Compute Optimizer は利用状況指標を分析して、リソースがアイドル状態かどうかを判断します。顧客はこの判定の閾値を設定できます。
