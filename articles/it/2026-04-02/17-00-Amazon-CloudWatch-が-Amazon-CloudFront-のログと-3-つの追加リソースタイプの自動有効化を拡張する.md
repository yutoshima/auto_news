# Amazon CloudWatch が Amazon CloudFront のログと 3 つの追加リソースタイプの自動有効化を拡張する

## メタデータ

- **情報源**: AWS What's New
- **カテゴリ**: it
- **公開日時**: 2026年04月02日 17:00
- **URL**: [https://aws.amazon.com/about-aws/whats-new/2026/04/amazon-cloudwatch-cloudfront-enablement/](https://aws.amazon.com/about-aws/whats-new/2026/04/amazon-cloudwatch-cloudfront-enablement/)

## 概要

<p>Amazon CloudWatchは現在、Amazon CloudFront Standard アクセスログ、AWS Security Hub CSPM ファインディングログ、および Amazon Bedrock AgentCore のメモリとゲートウェイのログとトレースを CloudWatch Logs に自動で有効化する機能をサポートしています。お客様は有効化ルールを設定して、既存および新規作成リソースの両方のテレメトリを自動的に構成できるようにし、手動設定なしで一貫した監視カバレッジを確保できます。</p> 
<p>有効化ルールは組織全体、特定のアカウント、または特定のリソースへスコープ設定することができます。</p>
