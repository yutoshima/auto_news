# CloudWatch Application Signalsが、エラー・パフォーマンスの異常・デプロイメントイベントを自動的に捕捉するようになりました

## メタデータ

- **情報源**: AWS What's New
- **カテゴリ**: it
- **公開日時**: 2026年07月06日 15:00
- **URL**: [https://aws.amazon.com/about-aws/whats-new/2026/06/cloudwatch-service-events/](https://aws.amazon.com/about-aws/whats-new/2026/06/cloudwatch-service-events/)

## 概要

本日、AWSは Amazon CloudWatch Application Signals のサービスイベントを発表します。これにより、追加のコード変更なしで、インストゥルメント済みサービスから例外とレイテンシーのイベントスナップショット、サービスレベルのパフォーマンスデータ、デプロイメントイベントを自動的に取得できます。顧客は CloudWatch コンソールの CloudWatch > Application Signals > [Service] > Errors に移動するだけで、デプロイメントが新たな例外を導入しているかどうかを迅速に特定できます。

Service Events は誰もが利用可能です
