# AWS Network Firewallは現在、Amazon EKSおよびAmazon ECS向けのコンテナ属性ベースの検査をサポートしています

## メタデータ

- **情報源**: AWS What's New
- **カテゴリ**: it
- **公開日時**: 2026年06月30日 17:00
- **URL**: [https://aws.amazon.com/about-aws/whats-new/2026/06/aws-network-firewall-container-attributes-referencing](https://aws.amazon.com/about-aws/whats-new/2026/06/aws-network-firewall-container-attributes-referencing)

## 概要

本日、AWSは、コンテナ化されたワークロードを保護する方法を簡素化する、AWS Network Firewallのコンテナ属性ベースのルールを発表します。これには、Amazon Elastic Kubernetes Service（Amazon EKS）および Amazon Elastic Container Service（Amazon ECS）で実行される生成系AIアプリケーションを含みます。これにより、Amazon EKS では名前空間、クラスター名、ラベルなどのネイティブなコンテナ構造を使ってファイアウォールポリシーを作成でき、Amazon ECS ではクラスター名とコンテナインスタンス属性を使用してポリシーを定義できるようになります。
