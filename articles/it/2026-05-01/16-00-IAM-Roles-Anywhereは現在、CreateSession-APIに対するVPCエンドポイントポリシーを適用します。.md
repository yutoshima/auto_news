# IAM Roles Anywhereは現在、CreateSession APIに対するVPCエンドポイントポリシーを適用します。

## メタデータ

- **情報源**: AWS What's New
- **カテゴリ**: it
- **公開日時**: 2026年05月01日 16:00
- **URL**: [https://aws.amazon.com/about-aws/whats-new/2026/05/iam-roles-anywhere-vpc/](https://aws.amazon.com/about-aws/whats-new/2026/05/iam-roles-anywhere-vpc/)

## 概要

<p><a href="https://aws.amazon.com/iam/roles-anywhere/">AWS Identity and Access Management (IAM) Roles Anywhere</a> は、IAM Roles Anywhere CreateSession API の VPC エンドポイント ポリシーを構成する機能を提供します。VPC エンドポイント ポリシーを更新して、CreateSession 操作を許可または拒否できます。CreateSession が VPC エンドポイント ポリシーの「許可」文に明示的に含まれていない場合、またはすべての操作を許可していない場合（例として「ro」を指定した場合など）、この操作は許可されません。
