# Amazon MQは現在、RabbitMQ 4のインプレース主要バージョンアップグレードをサポートしています。

## メタデータ

- **情報源**: AWS What's New
- **カテゴリ**: it
- **公開日時**: 2026年05月05日 16:50
- **URL**: [https://aws.amazon.com/about-aws/whats-new/2026/05/amazon-mq-inplace-upgrades-rabbitmq4/](https://aws.amazon.com/about-aws/whats-new/2026/05/amazon-mq-inplace-upgrades-rabbitmq4/)

## 概要

<p>Amazon MQ は現在、RabbitMQ ブローカーのインプレース版アップグレードをサポートしており、データを移行したり新しいブローカーを作成したりすることなく、RabbitMQ 4 へブローカーをアップグレードできます。これにより、RabbitMQ 3.13 から 4.2 へ、Amazon MQ コンソール、AWS CLI、または API から直接アップグレードできます。<br /><br /> インプレースアップグレードは、ブローカーの設定、キュー、エクスチェンジ、バインディング、ユーザー、ポリシーをそのまま維持します。RabbitMQ 4.2 では、クラシック・ミラーリングキューの削除など、後方互換性を損なう変更が導入されており、移行が含まれます。
