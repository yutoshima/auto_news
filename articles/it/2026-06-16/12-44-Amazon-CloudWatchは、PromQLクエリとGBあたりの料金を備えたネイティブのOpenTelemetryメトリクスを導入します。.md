# Amazon CloudWatchは、PromQLクエリとGBあたりの料金を備えたネイティブのOpenTelemetryメトリクスを導入します。

## メタデータ

- **情報源**: AWS What's New
- **カテゴリ**: it
- **公開日時**: 2026年06月16日 12:44
- **URL**: [https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-cloudwatch-otel-metrics/](https://aws.amazon.com/about-aws/whats-new/2026/06/amazon-cloudwatch-otel-metrics/)

## 概要

<p>Amazon CloudWatch は現在、OpenTelemetry メトリクスをネイティブにサポートしています。OpenTelemetry Protocol（OTLP）でメトリクスを送信し、Prometheus Query Language（PromQL）でクエリを実行でき、1GBあたりの取り込み料金と15か月分のストレージが含まれます。</p>
<p>これにより、カスタムアプリケーションのメトリクスと AWS 提供のメトリクスを、70を超えるサービスから単一のソリューションに統合し、PromQL で一括してクエリできます。CloudWatch は Prometheus 互換のクエリ API を提供しており、すでに OpenTelemetry を使用しているチームにも適しています。</p>
